## Proje Genel Bakış
Bu repo iki alt projeden oluşur:

1. **NeonRetro** (`frontend/`) — Siber-punk / 80s retro temalı demo e-ticaret sitesi. Tracker sistemini test etmek için kullanılır. Saf HTML + Vanilla JS, derleme adımı yok.
2. **Website & Advertisement Optimizer** (`Senior-Project-Website_Add_Optimizer/`) — Tracker backend'i ve admin dashboard'u. FastAPI (Python) backend + React dashboard.

## 1. Demo Mağaza (`frontend/`)

### Teknoloji
- HTML5 / Vanilla JavaScript
- Tailwind CSS (CDN)
- Font Awesome 6 (CDN)
- Browser localStorage (sepet ve favoriler)
- Stiller çoğunlukla sayfa içi `<style>` bloklarında; ortak ayrı CSS katmanı yok

### Kategori Mimarisi
Her kategori için ayrı HTML dosyası **yoktur** ve olmamalıdır. Tüm kategoriler tek bir dinamik sayfa üzerinden çalışır:

```
category.html?cat=electronics
category.html?cat=clothing
category.html?cat=beauty
category.html?cat=home-appliances
category.html?cat=books
category.html?cat=sports
```

Kategori ve ürün akışı artık `frontend/data/products.json` üzerinden beslenir. Yeni kategori ya da ürün seti eklenecekse önce veri üretim scripti / JSON şeması güncellenmeli; ayrı HTML sayfası açılmaz.

Mevcut kategoriler (`index.html`, `category.html`, `search.html` içinde tanımlı):
- `electronics` → Electronics
- `clothing` → Clothing
- `beauty` → Beauty
- `home-appliances` → Home Appliances
- `books` → Books
- `sports` → Sports

### Uygulama Durumu Notları
- Hedef mimari tek dinamik kategori akışı olsa da `frontend/` altında çok sayıda bağımsız HTML sayfası hâlâ mevcut; değişiklik yaparken eski kök dosya yollarını değil `frontend/` içeriğini referans al.
- Ürün listeleyen sayfalar (`index`, `category`, `product`, `search`, `favorites`, `new-arrivals`, `offers`) artık `frontend/data/products.json` dosyasından beslenir.
- `frontend/product.html` içinde attribute selector mantığı var; seçilen size/color/storage/skin type sepete ve tracker payload'ına eklenir.
- Sepet anahtarı `neon_cart`, favori anahtarı `neon_favorites`; tracker session'ı artık localStorage yerine bellek içi `sessionId` değişkeniyle tutuluyor.

## Frontend Data Architecture

**Single Source of Truth: `frontend/data/products.json`**
- Generated by `frontend/data/generate_products.py` from `Online Sales Data.csv`
- 232 unique products across 6 categories
- To regenerate: `python3 frontend/data/generate_products.py`

**Product Schema:**
```json
{
  "id": 1,
  "category": "electronics",
  "categoryName": "Electronics",
  "name": "iPhone 14 Pro",
  "price": 999.99,
  "image": "https://picsum.photos/seed/tech-1/400/400",
  "discount": 0,
  "salesCount": 87,
  "description": "...",
  "stock": 18,
  "rating": 3.6,
  "attributes": {
    "brand": "Apple",
    "storage": ["128GB", "256GB", "512GB", "1TB"],
    "colors": ["Space Black", "Silver", "Gold", "Deep Purple"]
  }
}
```

**Categories (6 total):**
| Slug | Display Name | Attribute Fields |

| beauty | Beauty | brand, skin_type, sizes (ml) |

## Tracker Akışı
**Full event payload sent to backend:**
```json
{
  "session_id": "session_k3x9mw2",
  "type": "click",
  "element": "add-to-cart",
  "page": "product",
  "product_id": 1,
  "category": "electronics",
  "price": 999.99,
  "brand": "Apple",
  "selected_size": null,
  "selected_color": "Space Black",
  "selected_storage": "128GB",
  "selected_skin_type": null
}
```

**session_id:** Generated in-memory on page load:  
`let sessionId = 'session_' + Math.random().toString(36).substr(2, 9);`  
No localStorage — sandbox/iframe compatible.

**Page names tracked (`tracker.js` getPageName()):**
- `product`, `cart`, `checkout`, `search`
- `electronics`, `clothing`, `beauty`, `home-appliances`, `books`, `sports`
- `home` (fallback)
